PART 7: AUTHENTICATION & CHAT PERSISTENCE (BACKEND IMPLEMENTATION)
=============================================================================
Status: COMPLETED

Goal: Make login functional and persist chats to user accounts

Implementation Steps (9 total):
-----------------------------------------------------------------------------

1. Create user storage with JSON file
   - File: backend/storage/users.py
   - UserStorage class with email lookup
   - Fields: id, email, password_hash, name, plan, created_at, updated_at
   - Data stored in: data/users.json

2. Add password hashing with bcrypt
   - File: backend/utils/auth.py
   - hash_password() - Hash password with bcrypt
   - verify_password() - Verify password against hash
   - Dependency: bcrypt>=4.0.0

3. Implement JWT token generation and validation
   - File: backend/utils/auth.py
   - create_access_token() - Generate JWT with user_id, email, expiry
   - decode_token() - Validate and decode JWT
   - Token expiry: 7 days (no refresh tokens for MVP)
   - Dependency: PyJWT>=2.8.0

4. Create auth API routes
   - File: backend/routes/auth.py
   - POST /api/auth/register - Create new user account
   - POST /api/auth/login - Login with email/password
   - GET  /api/auth/me - Get current authenticated user
   - POST /api/auth/logout - Logout (client-side token deletion)

5. Update auth middleware with real JWT validation
   - File: backend/middleware/auth.py
   - auth_required decorator - Require valid JWT
   - auth_optional decorator - Set user if token provided
   - get_current_user() - Get user from request context

6. Create chat/conversation storage
   - File: backend/storage/chats.py
   - ChatStorage class with user filtering
   - Fields: id, user_id, title, messages, pinned, is_tracking, etc.
   - Data stored in: data/chats.json

7. Create chat API routes
   - File: backend/routes/chats.py
   - GET    /api/chats - List user's chats
   - POST   /api/chats - Create new chat
   - GET    /api/chats/{id} - Get specific chat
   - PUT    /api/chats/{id} - Update chat
   - DELETE /api/chats/{id} - Delete chat
   - POST   /api/chats/{id}/messages - Add message to chat

8. Link tracking plans to users
   - File: burilar_api.py (updated)
   - Added user_id field to TrackingPlan class
   - Updated all tracking endpoints to filter by user_id
   - Uses auth_optional for backwards compatibility

9. Update frontend to use chat API
   - File: src/api/chats.ts (updated)
   - Fixed response format mapping (API -> frontend types)
   - File: src/App.tsx (updated)
   - Fetch chats from API when authenticated
   - Save/update/delete chats via API
   - Falls back to local state for unauthenticated users
